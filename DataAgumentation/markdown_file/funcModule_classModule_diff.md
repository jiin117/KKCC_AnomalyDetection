* 🏛️ 클래스 모듈
* 객체를 생성하기 위한 청사진(설계도)으로, 
* 데이터와 그 데이터를 처리하는 함수(메서드)를 하나의 단위로 묶습니다.
* 주요 장점:
* 상태 유지: 객체 생성 시 고유한 속성(데이터)과 상태를 메모리에 유지할 수 있습니다.
* 캡슐화 및 정보 은닉: 불필요한 데이터 노출을 막고, 객체 내부에서 데이터를 안전하게 조작합니다.
* 상속 및 다형성: 기존 클래스를 상속받아 기능을 확장하거나 수정하기 용이합니다.
* 주요 단점:
* 복잡성 증가: 상태 변화를 추적해야 하므로 디버깅이 까다로울 수 있습니다.
* 무거운 설계: 단순한 데이터 처리나 단일 기능 실행에는 오히려 코드가 길어지는 오버헤드가 발생합니다.
* 
* 🛠️ 함수 모듈
* 특정 작업을 수행하는 재사용 가능한 코드 블록들의 모음입니다. 
* 수학적 함수처럼 입력값(Input)을 받아 결과값(Output)을 반환하는 구조에 최적화되어 있습니다.
* 주요 장점:
* 높은 독립성: 입력값이 같으면 항상 같은 결과값(순수 함수)을 반환하므로 예측 가능하고 테스트가 쉽습니다.
* 단순하고 가벼움: 객체 생성 등 복잡한 초기화 과정이 없어 빠르게 작성하고 호출할 수 있습니다.
* 직관적인 구조: 절차적 프로그래밍이나 간단한 유틸리티(도구) 함수를 모아두기에 가장 적합합니다.
* 주요 단점:
* 상태 유지 불가: 함수가 종료되면 내부 데이터가 사라지므로, 
* 연속적인 데이터 처리를 위해서는 매번 값을 인자로 전달해야 합니다.
* 코드 중복: 유사한 데이터 세트를 다루는 여러 함수가 존재할 경우, 
* 데이터 구조 변경 시 관련된 모든 함수를 수정해야 하는 번거로움이 있습니다.
